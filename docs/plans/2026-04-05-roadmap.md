### 2.1 自定义策略编辑器

**目标**: 可视化构建交易策略规则

```

```

**策略结构**:
```typescript
interface CustomStrategy {
  id: string
  name: string
  conditionGroups: ConditionGroup[]  // 多个条件组
  actions: {
    type: 'buy' | 'sell' | 'hold'
    fraction?: number
  }[]
}
```

### 2.2 基准对比

**目标**: 对比策略 vs SPY/QQQ 基准，计算 Alpha/Beta/信息比率

- 权益曲线叠加显示
- Alpha = 策略年化收益 - 基准年化收益 × Beta
- Beta = Cov(策略收益, 基准收益) / Var(基准收益)
- IR = Alpha / Tracking Error

### 2.3 交易成本模型

**目标**: 精确模拟真实交易成本

```typescript
interface CostModel {
  commission: number       // 每笔固定佣金（美元）
  commissionRate: number   // 佣金比例（%）
  slippage: number         // 滑点（%）
  stampDuty: number        // 印花税（%）
  dividendReinvestment: boolean  // 分红再投资
}
```

### 2.4 深度风险分析

**目标**: 补充非常规风险指标

- **Omega 比率**: 收益分布高于阈值的概率加权和低于阈值的概率加权和之比
- **偏度 (Skewness)**: 收益分布的不对称性
- **峰度 (Kurtosis)**: 收益分布的肥尾程度
- **VaR (Value at Risk)**: 历史模拟法 vs 参数法对比
- **回撤事件分析**: 最大回撤持续时间、恢复时间、频率

### 3.1 因子有效性看板

**目标**: 每个因子展示历史 IC、IC_IR、分组收益单调性

```
src/components/factors/
├── FactorEffectiveness.tsx    # 有效性看板
├── IcChart.tsx               # IC 时序图
├── IcIrChart.tsx             # IC_IR 柱状图
├── GroupReturnChart.tsx      # 分组收益热力图
└── IcDecayChart.tsx          # IC 衰减图
```

**因子 IC 分析**:
```typescript
interface FactorEffectiveness {
  factorId: string
  name: string
  ic: {
    current: number
    history: { date: string; ic: number }[]
    mean: number
    std: number
  }
  icIr: number              // IC均值 / IC标准差
  groupReturns: {           // 按因子值分10组，各组收益
    group: number
    return: number
  }[]
  decayTest: {              // IC 衰减速度测试
    ic_1d: number
    ic_5d: number
    ic_20d: number
  }
}
```

**计算逻辑**:
- 选取宽基股票池（如 SPY 成分股或全市场前500）
- 计算每个因子与下期收益的 IC（Information Coefficient = Pearson相关系数）
- 按因子值排序分组，看收益是否单调递增

### 3.2 因子相关性热力图

### 3.3 预设策略模板

## 附录: 关键技术决策备忘

### 数据源优先级
- Yahoo Finance: 免费，速率限制 2req/s，数据完整但不稳定
- Finnhub: 免费层 60req/min，company 数据好，K线数据一般
- Polygon.io: 付费，数据最稳定，延迟低
- **决策**: 默认 Yahoo，失败时自动降级到 Finnhub，用户可手动配置优先级

### 策略编辑器评估
- 方案A (JSON UI): 灵活，但学习者仍需理解 JSON 结构
- 方案B (拖拽 UI): 更友好，但实现复杂
- **决策**: 先实现方案A（JSON + 表单结合），后续迭代拖拽

### 回测陷阱演示数据
- 使用真实历史数据 + 预设的"陷阱参数"
- 不需要模拟数据，直接用现有的 Yahoo 数据
